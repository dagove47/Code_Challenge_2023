Competition Requirements

  • Create a new public repository on Github.
  • Consistent commit progress in Github.
  • Share the repository URL in the registration form.
  • Repository submission deadline: November 10, 2023.
  • Animation and code presentation: November 17, 2023.
  • Time for project presentation: 15 to 20 minutes.
  • Animation theme: Animated Hero for Subaru, animated forms, animated helper character explaining a page section.
  • Maximum number of pages: 1 page.
  • Styles without external frameworks: The goal is to evaluate participants' abilities to write CSS and JS from scratch, so please do not use external frameworks like Bootstrap, Vue.js, etc.
  • Comments and readability: Participants are requested to use appropriate comments in their code to explain complex sections or design decisions.
  • Technologies to Use:
    • HTML.
    • CSS or all CSS pre-processors.
    • Vanilla Javascript.
  • HTML:
    • Semantic HTML structure: Use semantically appropriate HTML tags to ensure a clear and accessible animation structure.
    • Specific attributes and properties: Specify the use of certain HTML attributes or properties, such as the data- attribute, to facilitate communication between HTML and JavaScript.
  • Vanilla Javascript (Recommendations):
    • Events and Listeners: Use event listeners to detect and respond to user events, such as clicks or mouse movements.
    • DOM Manipulation: Use JavaScript to manipulate the DOM and create or modify elements necessary for the animation.
    • Performance Optimization: Encourage participants to optimize their JavaScript code to avoid unnecessary loops, excessive resource usage, or redundant code.
  • Compatibility and Responsive Design: The animation must be compatible with different browsers and devices, and it should have a responsive design that adapts to various screen sizes.
  • Evaluation and Criteria:
    • Establish criteria for evaluating animations, which may include:
      • Originality.
      • Visual quality.
      • Code efficiency.
      • Complexity of the animation.
      • Functionality.
      • Among other factors.
  • Each project must be completed individually.

GitHub Pages

To deploy your repository to GitHub Pages using GitHub:

  1. On GitHub, navigate to your repository's Settings page.
  2. In the Code and automation section of the sidebar, click Pages.
  3. Under Source, select Deploy from a branch.
  4. Under Branch, select the branch that contains your website files.
  5. Click Save.

Your website will be published to https://username.github.io/repository_name. It may take a few minutes for the changes to propagate.
